Pulse Eval Guide

Pulse has three eval layers: free deterministic checks that run automatically, a quality eval suite with an LLM judge, and an optional A/B comparator for model swaps. Start from the top — run the free checks first, fix anything they catch, then move to the quality suite.

Typical Workflow

1Run unit tests

After any code change, run npm test. These are instant and free. Fix anything before moving on.

2Start the server

Run PULSE_TEST_MODE=true PULSE_NO_RATE_LIMIT=true node src/server.js and wait for the event cache to load.

3Smoke test in the simulator

Open /test and try a few neighborhoods. Confirm the change works manually before running eval suites.

4Run quality evals

Run npm run eval:quality to score all 15 golden conversations on 6 dimensions. Review results at /eval-quality.

5Check extraction & scrape audits

Open /health to check extraction pass rates and source coverage. These run on every scrape.

6Review traces

Open /eval to browse recent traces. Code evals (char limit, valid picks, etc.) run automatically on each trace.
