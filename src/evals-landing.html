Pulse Eval Guide

Pulse has seven evaluation layers, from free deterministic checks to LLM-judged conversation tests. Start from the top — run the free checks first, fix anything they catch, then move down to the costlier layers. Each card below shows the live pass rate (when available), what it costs, and how to run it.

Unit Tests
Free
Pure function correctness — formatters, geo math, filter logic, session helpers. No server needed. Run these first after any code change.

Typical Workflow

1Run unit tests

After any code change, run npm test. These are instant and free. Fix anything before moving on.

2Start the server

Run npm run dev and wait for the event cache to load (~3 min on first boot, instant on restart).

3Smoke test in the simulator

Open /test and try a few neighborhoods. Confirm the change works manually before running eval suites.

4Run targeted regressions

If your change affects a specific principle, run just that principle's tests: --principle P1.

5Run the full eval suite

Run scenario evals to validate end-to-end behavior. Review failures in the Reports dashboard.

6Label scenarios

In the Reports dashboard, review conversations and mark pass/fail. Use j/k to navigate, p/f to label quickly.
